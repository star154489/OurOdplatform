## 🚀 快速启动

### 1. 环境准备

```bash
# 进入项目根目录
cd ODPlatform/

# 安装 od_platform 核心包（开发模式）
pip install -e apps/platform/

# 安装 Web 额外依赖
pip install -r web/requirements.txt
```

> **依赖说明**：
> - `fastapi` + `uvicorn` — Web 框架
> - `python-multipart` — 文件上传支持
> - `ultralytics` + `torch` + `opencv-python` — 推理核心（与 od_platform 共用）
> - 完整依赖列表见 `web/requirements.txt`

### 2. 启动后端服务

```bash
# 在项目根目录执行
python web/backend.py
```

启动成功后输出：
```
项目根目录: F:\wok\OurOdplatform
前端目录:   F:\wok\OurOdplatform\web\frontend
源文件路径: F:\wok\OurOdplatform\apps\platform\src
模型加载完成: ..., 类别数: 80
启动 Web 服务: http://localhost:8000
```

### 3. 打开前端页面

浏览器访问：**http://localhost:8000**

## 📡 API 接口说明

### `GET /` — 前端页面
返回目标检测 Web 操作界面。

### `GET /api/health` — 健康检查
```json
{
  "status": "ok",
  "model_loaded": true,
  "model_name": "yolo11n.pt",
  "device": "cuda:0"
}
```

### `GET /api/models` — 获取可用模型列表
返回项目中所有可用的 `.pt` 模型文件及其大小。

### `POST /api/detect` — 图片目标检测

**请求**：`multipart/form-data`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `files` | File[] | ✅ | 图片文件（可多选，支持 JPG/PNG/BMP/WebP） |
| `model` | string | ❌ | 模型文件名，默认 `yolo11n.pt` |
| `conf` | float | ❌ | 置信度阈值 (0-1)，默认 0.25 |
| `custom_hint` | string | ❌ | 自定义提示文字 |

**响应**：
```json
{
  "success": true,
  "results": [
    {
      "filename": "example.jpg",
      "original": "data:image/jpeg;base64,/9j/4AAQ...",
      "annotated": "data:image/jpeg;base64,/9j/4AAQ...",
      "detections": [
        {
          "label": "person",
          "confidence": 0.9542,
          "box": [120, 80, 400, 550]
        }
      ],
      "count": 1,
      "hint": "系统提示: 共检测到 1 个目标 (person×1)。",
      "width": 640,
      "height": 480
    }
  ],
  "total": 1,
  "total_detections": 1,
  "model_used": "yolo11n.pt",
  "conf_threshold": 0.25
}
```

## 🎯 使用说明

### 单张图片识别
1. 点击 **"选择图片文件"** 按钮，选择一张本地图片
2. （可选）调整置信度阈值、输入自定义提示文字
3. 点击 **"开始识别"**
4. 查看识别结果：左侧原图 | 右侧检测效果图 | 下方检测列表

### 文件夹批量识别
1. 点击 **"选择图片文件夹"** 按钮，选择一个包含图片的本地文件夹
2. 系统自动加载文件夹内所有支持的图片格式
3. 点击 **"开始识别"**，后端逐个处理
4. 结果列表支持**分页浏览**（每页 5 张）

### 拖拽上传
- 直接将图片文件或图片文件夹**拖拽到上传区域**即可

### 模型切换
- 在下拉菜单中选择不同模型（如 `yolo26n.pt`、`best.pt`）
- 切换模型后首次推理会自动重新加载

### 置信度调整
- 调高（0.5+）：减少误检，适合需要精确结果的场景
- 调低（0.1-0.25）：减少漏检，适合需要全面检测的场景

  -F "files=@test.jpg" \
  -F "conf=0.25" \

  -F "files=@img1.jpg" \
  -F "files=@img2.jpg" \
  -F "files=@img3.jpg" \
  -o result.json

## 📂 目录兼容性说明

- 所有路径使用**相对路径**，通过 `.odp-workspace` 标记文件自动定位项目根目录
- 整套 `web/` 文件夹可随项目整体**复制到任意磁盘目录**运行
- 模型搜索路径（按优先级）：项目根目录 → `models/pretrained/` → `models/trained/` → `epoch/`
- 若模型文件未找到，ultralytics 会自动从网络下载（需要网络连接）

## ❓ 常见问题

**Q: 启动报错 `找不到 .odp-workspace 标记文件`**
A: 请确保在 ODPlatform 项目根目录下执行 `python web/backend.py`。

**Q: 模型加载很慢**
A: 首次加载 YOLO 模型需要初始化 PyTorch 和加载权重，通常 5-30 秒。后续推理很快。

**Q: 没有 GPU 怎么办**
A: 自动降级到 CPU 推理，速度较慢但功能正常。CPU 推理一张图约 0.5-3 秒。

**Q: 上传大图浏览器卡顿**
A: 后端自动将图片长边缩放到 1920px 以内，避免超大图片传输问题。可修改 `MAX_IMAGE_DIM` 调整。

**Q: 文件夹上传选择后没有加载所有图片**
A: 浏览器文件夹选择仅加载当前目录下的文件，不含子目录。支持格式：jpg, jpeg, png, bmp, tiff, tif, webp。
